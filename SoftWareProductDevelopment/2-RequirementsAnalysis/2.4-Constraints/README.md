# 2.4 Constraints (Обмеження системи)

У цьому розділі наведено технічні, програмні та організаційні обмеження, які необхідно враховувати під час розробки системи.

## 1. Технічні обмеження
- Система повинна коректно працювати у веб-браузерах Google Chrome, Mozilla Firefox, Safari, Microsoft Edge.
- Рекомендована роздільна здатність екрана користувача — не менше 1280×720 px.
- Сервер повинен підтримувати REST API та HTTPS-з'єднання.

## 2. Програмні обмеження
- Використання мови програмування: Java / JavaScript / Python (у межах навчального прикладу).
- База даних: PostgreSQL або інша сумісна SQL-система.
- Заборонено використання пропрієтарних (закритих) API без відповідних ліцензій.

## 3. Організаційні обмеження
- Доступ користувачів до системи обмежений ролями (студент, дослідник, адміністратор).
- Всі зміни вносяться тільки після перевірки викладачем або адміністратором.
- Дотримання академічної доброчесності при роботі з науковими ресурсами.

## 4. Безпекові обмеження
- Паролі користувачів зберігаються у хешованому вигляді.
- Усі запити до сервера повинні виконуватись через HTTPS.
- Доступ до сторонніх сервісів (Google Scholar, ResearchGate тощо) — тільки через офіційні API.
